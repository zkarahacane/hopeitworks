## Décisions Architecture FINALES (validées par Zakari)

| Composant | Choix |
|-----------|-------|
| Backend | Go (chi + sqlx + oapi-codegen, structure handler/service/repository) |
| Frontend | Vue 3 + TypeScript strict + Vite (SPA, pas Nuxt) |
| CLI | Go |
| Auth MVP | JWT maison OIDC-compatible (~200 lignes Go), switch Keycloak = 1 config change |
| DB | Postgres (state + events via LISTEN/NOTIFY) |
| Events | Postgres LISTEN/NOTIFY (pas Redis) |
| Streaming UI | SSE (pas WebSocket), actions user = POST REST |
| Repo | Monorepo (api/, web/, cli/, agent/, deploy/, migrations/) |
| OpenAPI | Spec-first avec oapi-codegen (api.yaml = source de vérité) |
| Container runtime dev | Docker socket mount + docker-socket-proxy (tecnativa) |
| Container runtime prod | K8s Jobs |
| Logs | slog JSON (stdlib Go 1.21+) |
| Secrets agents | Env vars injectées à la création du container |
| Cost control | Timeout hard 30min + compteurs DB daily + circuit breaker 3 failures |
| Versioning | Semantic versioning, conventional commits |

## Principes Clés

1. CI reste sacrée (Hopeworks s'adapte autour, ne remplace pas)
2. Event-driven via Postgres LISTEN/NOTIFY
3. Agents Claude Code dans Docker containers (--dangerously-skip-permissions, safe car isolé)
4. Stories lightweight < 2KB (pointeurs, agents enrichissent le contexte)
5. Spec-first OpenAPI (agents modifient la spec, d'autres implémentent)
6. Web-first + CLI day-1 (les deux first-class)
7. Token tracking mandatory, cost limits optional
8. Human gates configurables (auto-merge docs, require approve code)
9. Pause/resume même en mode auto
10. Dogfooding dès que possible (scripts bash en attendant v0.4.0)

## Learnings Zakari (Design Constraints)

- Trop de contexte pollue → stories courtes, contexte précis
- Tests E2E auto = primordial
- Session main ne doit pas se polluer → agents dans containers
- Paralléliser quand possible
- Env recette local → cloud si budget
- Back/Front séparés dans le monorepo pour contexte agent isolé
- Mock data critique (todo app + seed SQL)
- Claude main = orchestrateur, ne code pas, délègue
