# Story R-2-2: [BACK] Backend action: git_pr

Status: ready-for-dev

## Story

As a **pipeline executor**,
I want a `git_pr` action that opens a pull request from the current feature branch,
so that the pipeline can automate the code review handoff without manually triggering `gh pr create`.

## Acceptance Criteria (BDD)

### Scenario 1: PR created with rendered title and branch from metadata

```gherkin
Given a pipeline step with action_type "git_pr" and config:
  | title_template | feat({scope}): {story_title}  |
  | target_branch  | main                          |
  | draft          | false                         |
And RunContext.Metadata["branch_name"] is "feat/S-03-add-login-page"
When the action executes
Then GitProvider.CreatePR is called with:
  | workDir    | value from Metadata["work_dir"]       |

  And the action returns nil
```



## Dev Notes

### Dependencies

- **R-1-3 (action_types + step config plumbing) — required:** `git_pr` must be in the action type enum and `RunStep.Config` must be populated by the executor.
- **R-2-1 (git_branch action) — logical prerequisite:** In a pipeline, `git_branch` runs before `git_pr` and writes `Metadata["branch_name"]`. This action reads that key.
- **Story 3-3 (GitProvider PR operations) — DONE:** `GitProvider.CreatePR` is already implemented via `gh pr create`.

### Architecture Requirements

- `GitPRAction` is an adapter in `backend/internal/adapter/action/`. It implements `model.Action`.
- No direct coupling to the GitHub adapter — only the `port.GitProvider` interface is used.
- The PR body is generated purely in Go; no template engine needed for this simple case.
- The `work_dir` is the path to the cloned repository on the host (passed via metadata or step config from the `agent_run` or a clone step).

### Technical Specifications

**File:** `backend/internal/adapter/action/git_pr.go`

```go
package action

import (
    "context"
    "fmt"
    "log/slog"
    "strings"

    "github.com/zakari/hopeitworks/backend/internal/domain/model"
    "github.com/zakari/hopeitworks/backend/internal/domain/port"
)

// GitPRAction implements model.Action for creating a GitHub pull request.
// It renders the PR title from a configurable template using story context,
// uses the branch_name from RunContext.Metadata as the source branch,
// and stores the resulting PR URL in RunContext.Metadata["pr_url"].
type GitPRAction struct {
    gitProvider port.GitProvider
    storyRepo   port.StoryRepository
    logger      *slog.Logger
}

func NewGitPRAction(gitProvider port.GitProvider, storyRepo port.StoryRepository, logger *slog.Logger) *GitPRAction {
    return &GitPRAction{gitProvider: gitProvider, storyRepo: storyRepo, logger: logger}
}

func (a *GitPRAction) Name() string { return "git_pr" }

func (a *GitPRAction) Execute(ctx context.Context, runCtx *model.RunContext) error {
    cfg := runCtx.RunStep.Config

    titleTemplate, ok := cfg["title_template"]
    if !ok || titleTemplate == "" {
        titleTemplate = "{story_key}: {story_title}"
    }
    targetBranch, ok := cfg["target_branch"]
    if !ok || targetBranch == "" {
        targetBranch = "main"
    }
    draft := cfg["draft"] == "true"

    workDir, err := resolveWorkDir(cfg, runCtx.Metadata)
    if err != nil {
        return err
    }

    branchName, ok := runCtx.Metadata["branch_name"].(string)
    if !ok || branchName == "" {
        return fmt.Errorf("git_pr: branch_name not found in run context metadata — ensure git_branch step ran first")
    }

    story, err := a.storyRepo.GetByID(ctx, runCtx.StoryID)
    if err != nil {
        return fmt.Errorf("fetch story: %w", err)
    }

    scope := ""
    if story.Scope != nil {
        scope = *story.Scope
    }

    title := renderTemplate(titleTemplate, map[string]string{
        "story_key":   story.Key,
        "story_title": story.Title,
        "scope":       scope,
        "branch_name": branchName,
    })
    if draft {
        title = "[Draft] " + title
    }

    body := buildPRBody(story)

    a.logger.Info("creating pull request",
        "branch", branchName, "target", targetBranch,
        "story_key", story.Key, "draft", draft)

    prURL, err := a.gitProvider.CreatePR(ctx, workDir, title, body, targetBranch)
    if err != nil {
        return fmt.Errorf("create PR: %w", err)
    }

    runCtx.Metadata["pr_url"] = prURL
    a.logger.Info("pull request created", "pr_url", prURL)
    return nil
}

// renderTemplate replaces {key} placeholders in tpl with values from vars.
func renderTemplate(tpl string, vars map[string]string) string {
    result := tpl
    for k, v := range vars {
        result = strings.ReplaceAll(result, "{"+k+"}", v)
    }
    return result
}

// buildPRBody constructs a standard PR body from story context.
func buildPRBody(story *model.Story) string {
    var b strings.Builder
    b.WriteString("## " + story.Key + ": " + story.Title + "\n\n")
    if story.Objective != nil && *story.Objective != "" {
        obj := *story.Objective
        if len(obj) > 500 {
            obj = obj[:500] + "…"
        }
        b.WriteString("### Objective\n\n" + obj + "\n\n")
    }
    b.WriteString("---\n> Generated by hopeitworks pipeline")
    return b.String()
}
```

**Shared helper** (`resolveWorkDir`) — to avoid duplication between `git_branch.go` and `git_pr.go`, extract into `backend/internal/adapter/action/helpers.go`:

```go
// resolveWorkDir reads work_dir from step config, falling back to metadata.
func resolveWorkDir(cfg map[string]string, meta map[string]any) (string, error) {
    if wd, ok := cfg["work_dir"]; ok && wd != "" {
        return wd, nil
    }
    if wd, ok := meta["work_dir"].(string); ok && wd != "" {
        return wd, nil
    }
    return "", fmt.Errorf("work_dir not configured in step config or metadata")
}
```

**Metadata contract:**
- Reads: `branch_name` (required, set by `git_branch`), `work_dir` (optional fallback)
- Writes: `pr_url` (string — the full GitHub PR URL)

**Error codes produced:**
- `STORY_NOT_FOUND` — from StoryRepository
- `GIT_PR_FAILED` — GitProvider.CreatePR returned error (wrapped)
- Missing `branch_name` — user-facing error message describes missing prerequisite
